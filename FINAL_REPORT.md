## Project Summary

SyncthingTray is a lightweight Windows system tray manager for Syncthing. It launches syncthing.exe hidden (no console window), provides start/stop/restart controls from the tray menu, and shows sync status in the tooltip. Built as a single AutoHotkey v2 script (~280 lines), it uses INI-based settings, a dark-themed Settings GUI, and communicates with Syncthing's REST API for graceful shutdown and sync status polling.

## Key Features

- Launches Syncthing hidden (no console window) with `--no-browser` flag
- Tray icon reflects running state (sync/pause icons with shell32.dll fallbacks)
- Start, Stop, and Restart controls from the tray menu
- Open Syncthing Web UI (localhost:8384) with double-click or menu item
- Dark-themed Settings GUI: double-click behavior, run at startup, API key configuration
- Graceful shutdown via REST API (`POST /rest/system/shutdown`) with ProcessClose fallback
- Sync status indicator in tooltip: idle / syncing (N%) / error / stopped (polls every 10s)
- Crash detection: TrayTip alert + SoundBeep when Syncthing exits unexpectedly
- Windows startup shortcut via `FileCreateShortcut` (works compiled or as .ahk)
- Periodic state sync (5-second polling timer) to keep tray icon accurate
- GitHub link in Settings GUI

## Lessons Learned

### AHK REST API via WinHttp
AHK v2 can make HTTP requests using `ComObject("WinHttp.WinHttpRequest.5.1")`. This is a synchronous COM call — do not use in the main thread if you need responsiveness. For tray apps, use SetTimer + a global state variable to poll async-style.

### Intentional Stop Pattern
When a tray app wraps a process that can also crash, track intentional stop state separately from unexpected exit. Use a global boolean (`g_intentionalStop`) that is set true before any manual stop/restart, then reset after the crash detection runs. This prevents false "crashed!" alerts on user-initiated stops.

### AHK Graceful Shutdown with Polling
After sending a REST shutdown request, use a polling loop (`loop 50 { if !ProcessExist(...) return; Sleep(100) }`) to wait up to 5 seconds for clean exit before falling back to `ProcessClose`. This pattern works for any process that supports an API-triggered shutdown.

### Tray Menu Default Item
In AHK v2, the default tray menu item (bold text, double-click target) is set with `tray.Default := "ItemName"`. To unset it (no default/bold), use `try tray.Default := ""` wrapped in try/catch — AHK may throw if the item text is empty.

### Settings GUI Single-Instance Guard
Use `if WinExist("WindowTitle") { WinActivate(...); return }` at the start of any settings window handler to prevent duplicate windows. This is simpler than tracking a global GUI reference.

### startup Shortcut: Compiled vs Script
`FileCreateShortcut` must target the .exe when compiled and `A_AhkPath` (with the script as args) when running as source. Check `A_IsCompiled` to branch:
```ahk
target := A_IsCompiled ? A_ScriptFullPath : A_AhkPath
args   := A_IsCompiled ? "" : '"' A_ScriptFullPath '"'
```

### INI File Security
If the INI file may contain an API key (or any credential), ensure it is in `.gitignore` (`*.ini` pattern covers all INI files). Document this prominently in CLAUDE.md and README.
